# Техническое задание: Университетский календарь

## 1. Общее описание проекта

Разработать веб-приложение коллективного календаря для студентов университета, позволяющее управлять расписанием занятий, отслеживать дедлайны домашних заданий и экзаменов, а также совместно работать с файлами и заметками.

## 2. Цели и задачи

### Основная цель
Создать единую платформу для организации академической деятельности студентов с возможностью совместного редактирования и обмена материалами.

### Задачи
- Обеспечить удобное планирование и отслеживание учебных событий
- Реализовать систему совместной работы с календарем
- Организовать централизованное хранение и обмен учебными материалами
- Предоставить инструменты для отслеживания дедлайнов и подготовки к экзаменам

## 3. Функциональные требования

### 3.1 Управление календарем
- Просмотр календаря в режимах: месяц, неделя, день
- Создание, редактирование и удаление событий
- Навигация по датам с интуитивным интерфейсом
- Поиск событий по названию, курсу, типу

### 3.2 Типы событий
- **Расписание занятий**: регулярные еженедельные пары
- **Домашние задания**: дедлайны с указанием приоритета
- **Экзамены**: даты экзаменов и зачетов
- **Прочие события**: дополнительные академические мероприятия

### 3.3 Детализация событий
Каждое событие должно содержать:
- Название
- Описание
- Курс/предмет
- Дата и время начала/окончания
- Аудитория/место проведения
- Преподаватель
- Уровень приоритета (низкий, средний, высокий)
- Прикрепленные файлы

### 3.4 Повторяющиеся события
- Настройка еженедельного расписания занятий
- Возможность указать дни недели для повторения
- Установка периода действия расписания
- Исключения для каникул и праздников

### 3.5 Работа с файлами
- Прикрепление файлов к событиям (конспекты, фото, презентации)
- Поддержка форматов: PDF, DOC, DOCX, JPG, PNG, PPT, PPTX
- Предварительный просмотр файлов
- Скачивание и совместное использование материалов
- Организация файлов по курсам и типам событий

### 3.6 Совместная работа
- Возможность редактирования календаря несколькими пользователями
- Синхронизация изменений в реальном времени
- Отображение автора создания/изменения события
- Разрешение конфликтов при одновременном редактировании

### 3.7 Уведомления и напоминания
- Настраиваемые напоминания о дедлайнах
- Уведомления о предстоящих занятиях
- Оповещения о добавлении новых файлов
- Напоминания о подготовке к экзаменам

## 4. Нефункциональные требования

### 4.1 Производительность
- Загрузка календаря не более 2 секунд
- Синхронизация изменений в реальном времени
- Поддержка файлов размером до 50 МБ
- Стабильная работа при одновременном использовании 50+ пользователей

### 4.2 Удобство использования
- Интуитивно понятный интерфейс
- Адаптивный дизайн для мобильных устройств
- Возможность работы без подключения к интернету (просмотр)
- Быстрое создание событий (не более 3 кликов)

### 4.3 Безопасность
- Аутентификация пользователей
- Разграничение прав доступа
- Защищенная передача данных
- Резервное копирование информации

### 4.4 Совместимость
- Работа в современных веб-браузерах
- Корректное отображение на мобильных устройствах
- Возможность экспорта в популярные календарные приложения

## 5. Пользовательские роли

### 5.1 Студент
- Просмотр календаря
- Создание и редактирование событий
- Загрузка и скачивание файлов
- Получение уведомлений

### 5.2 Администратор группы
- Все права студента
- Модерация событий
- Управление доступом к календарю
- Настройка общих параметров

## 6. Интерфейс пользователя

### 6.1 Главная страница
- Календарная сетка с событиями
- Панель навигации по датам
- Кнопка создания нового события
- Фильтры по типам событий и курсам

### 6.2 Форма создания события
- Поля для ввода основной информации
- Настройки повторения для регулярных занятий
- Область для загрузки файлов
- Выбор уровня приоритета

### 6.3 Просмотр события
- Отображение всех деталей события
- Список прикрепленных файлов с возможностью просмотра
- Информация об авторе и времени создания
- Кнопки редактирования и удаления

### 6.4 Мобильная версия
- Адаптированная навигация
- Упрощенные формы ввода
- Оптимизированный просмотр файлов
- Поддержка жестов для навигации

## 7. Интеграции

### 7.1 Экспорт данных
- Экспорт календаря в формат iCal
- Синхронизация с Google Calendar
- Выгрузка списка событий в Excel

### 7.2 Уведомления
- Email-уведомления о важных событиях
- Push-уведомления в браузере
- Интеграция с мессенджерами (опционально)

## 8. Критерии приемки

### 8.1 Функциональность
- Все типы событий создаются и отображаются корректно
- Файлы успешно загружаются и открываются
- Изменения синхронизируются между пользователями
- Повторяющиеся события работают согласно настройкам

### 8.2 Производительность
- Календарь загружается за время не более 2 секунд
- Файлы до 50 МБ загружаются без ошибок
- Приложение стабильно работает при 50 одновременных пользователях

### 8.3 Удобство использования
- Новый пользователь может создать расписание семестра за 30 минут
- Мобильная версия обеспечивает полную функциональность
- Интерфейс интуитивно понятен без дополнительного обучения

## 9. Ограничения и предположения

### 9.1 Ограничения
- Максимальный размер файла: 50 МБ
- Поддержка браузеров не старше 2 лет
- Требуется подключение к интернету для синхронизации

### 9.2 Предположения
- Пользователи имеют базовые навыки работы с веб-приложениями
- Доступ к интернету стабилен в учебных заведениях
- Студенты заинтересованы в совместной работе с календарем
